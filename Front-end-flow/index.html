Front-End Flow Documentation

1. User Registration Flow

When a user logs in for the first time, the system will check if they have a sharable link generated.

  • If sharableLink is false in the backend response, the frontend will show an option to "Create Sharable Link".
  • If the user already has a sharable link, they will be redirected to the Dashboard where they can manage sessions.
// Sample Backend Response
{
"message": "User Fetched",
"sharableLink": false, // New User
}

{
"message": "User Fetched",
"sharableLink": "http://example.com/sharable?userId", // Existing User
}

2. Existing User Dashboard

If the user is already registered and has a sharable link, the UI will show:

  • Dashboard - Overview of the userâ€™s activities.
  • Events - Displays all upcoming and completed events.
  • Edit Availability - Provides the option to update time slots for booking.

3. Edit Availability Flow

If the user selects Edit Availability:

  • Redirect to the Edit Slots Page.
  • Display a Calendar, allowing the user to select a date.
  • Upon selecting a date, a Modal will appear where the user can choose available timings.
  • The user can either add new slots or replace existing slots.
  • Send the updated slots to the server and update the backend availability.

4. Creating a Sharable Link

If the user is new and does not have a sharable link, the UI will provide an option to Create Sharable Link. Once created, this link allows others to book sessions.

Create Sharable Link

5. Booking a Session via Sharable Link

Once the sharable link is created, another user can open the link and book a session. The steps include:

  • Open the link and view the host's availability.
  • Select a date from the calendar.
  • Choose from available time slots for that date.
  • Confirm the booking and submit the form to schedule the session.
